Compatibility

We’ve shown you some of the most innovative technologies you can integrate into your email marketing campaigns. But how do they actually perform? Which email clients support them best? And what happens when they’re not supported?

The good news is that these technologies deliver outstanding performance in premium environments. On desktop via web browsers (view online), and across Apple devices — both desktop and mobile (iMail, Outlook for Mac) — the experience is simply seamless.
From interactive carousels to hover effects, users on these devices can enjoy a dynamic, engaging experience that brings your content to life.

Ensure that only one of these components is used at a time; this is to prevent conflicts arising from CSS inheritance, which could compromise their correct functioning.

And what about the rest?
On Windows Outlook and Android mobile devices, content is still displayed effectively thanks to fallback images — ensuring your message remains clear, consistent, and impactful in every scenario.

Check out the following table to see if an email client supports CSS animations.

Carousel & HotSpot:

we recommend using a single fallback image, even if the component contains three or more images; the fallback image should summarise all the images of the components .
Why a single image? These components require a very long piece of code to be created, and this code may cause the email to be clipped in Google, especially if the email content is lengthy. For HotSpot you can use an additional image for mobile.

Video Player:

to display a video in an email, you need a video file hosted on a CRM (if possible) or on the client’s website; unfortunately, it is not possible to use a YouTube link or ID for direct playback. The link must be something like: https://www.yourwebsite.com/video/filename.mp4.

Videos in marketing emails should be kept very small to ensure fast loading and high deliverability. Use a thumbnail image (under 100–200 KB) that links to the video hosted on an external site. If embedding directly, keep files under 10 MB, with a duration of 30–60 seconds and a width of around 600–900 pixels.

You will need a poster image, which is the image people will see in the email while the video loads. Your poster image should be a still from the video with a play icon overlaid, so users know it is a video. You can also use this image to include instructions on how to play the video. After creating the image, upload it to your server or ESP.

Accordion, CSS Animations & Hamburger Menu:

as the functionality is not supported in Outlook and Gmail, the entire accordion content will be displayed as static.
